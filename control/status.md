When the key expires it is stale for one recompute window T; under a plain TTL cache WITHOUT coordination every request arriving in [0,T) independently recomputes it at the origin, so the per-expiry origin herd is exactly Poisson(λT) — expected size λT — which GROWS LINEARLY with request rate (the Wikipedia example verbatim: 10 req/s × 3 s render = 30 concurrent recomputes; a hotter key stampedes harder).
