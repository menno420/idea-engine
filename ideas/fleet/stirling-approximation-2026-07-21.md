## Method

Exact rational backbone first, Monte-Carlo agreement and high-precision bracket second.

**Exact backbone (`fractions.Fraction`, no floats).** `bernoulli(upto)` builds `B_0..B_upto` from the recurrence `Σ_{k=0}^{m} C(m+1,k)B_k = 0`; `log_series_coeffs` returns `c_k = B_{2k}/(2k(2k−1))`; `ratio_series_coeffs` builds `a_0..a_6` from the log-series by the exp-of-series recurrence `a_k = (1/k)Σ_{j=1}^k j·l_j·a_{k−j}` (with `l_j` the log-series coefficient, nonzero only for odd `j`); `recover_log_from_ratio` inverts it (log-of-series) and must return the `l_j` back. G1 asserts the Bernoulli values `B_0..B_14`, the `c_k`, the `a_k`, AND the exp→log round-trip all match with zero tolerance — `checked = 29`, `mismatches = 0`. The limit value `1` is trivial; the teeth are the exact rational corrections.

**Monte-Carlo integral (Gamma importance sampling).** `mc_ratio(rng, n, shape, draws)` estimates `n! = ∫_0^∞ x^n e^{−x} dx` by importance sampling from a `Gamma(shape, 1)` proposal (stdlib `random.gammavariate`). The proposal's normalizer is the **exact integer** `Γ(shape)` — for `shape = n` this is `(n−1)!`, the proposal's own normalization, **not** the answer `n!` — so the per-draw weight `w(x) = Γ(shape)·x^{n−shape+1}` is a genuine unbiased estimator of `n!` (for `shape = n`, `E[(n−1)!·x] = (n−1)!·n = n!`). The ratio `r̂ = n!/D`, `D = √(2πn)(n/e)^n`, is formed in log-space (`C = Γ(shape)/D` via `lgamma`) so no astronomically large integer or float ever materializes. Each draw is **independent**, so the plain iid standard error is the honest one — no batch means / thinning (those are only for autocorrelated sample paths, which this is not). `random.seed(SEED)` is re-seeded at the START of each MC gate so gate order cannot perturb the payload.

**Robbins bracket (high-precision `Decimal`).** `ln(r_n) = ln(n!) − (½ln(2πn) + n·ln n − n)` is evaluated at `prec = 60` with a 60-digit `π`, and compared against the exact rational bounds `1/(12n+1)` and `1/(12n)` for `n = 1..64`. The bound expressions are exact rationals; the log is computed to 60 digits and lands strictly inside every time.
