## 💡 Session idea

**Teach the upgrade's carve-out banking to certify "template unchanged —
banked copy is restorable verbatim."** Observed this session: after the regen
dropped the enabler's two host customizations, `diff` of the pre-regen bank
(sha e3275f45) against the regenerated file contained ONLY the host hunks —
the v1.15.0 enabler template was byte-identical to v1.10.0's for this config —
so the byte-faithful re-apply was provably a straight `cp` of the bank, and
the enabler landed as a net git no-op. The kit already computes everything
needed at bank time: it could compare regen output vs (bank minus detected
host steps) and, on equality, print `carve-out: template unchanged — restoring
the banked copy verbatim is exact` (or auto-restore behind a flag), turning
the riskiest manual step of every upgrade (hand-splicing host steps into an
evolved template, as the gate DID require this time) into a certified fast
path whenever the template didn't move. Slice-sized: the comparison is one
normalized diff inside the existing carve-out scan; the gate/enabler asymmetry
this session (gate template evolved, enabler didn't) is the exact pair of test
fixtures.
