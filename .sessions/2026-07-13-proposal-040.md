## 💡 Session idea

**Make `check_ideas.py --outbox` resolve this-repo SHA pins against local git —
a typo'd blob SHA currently ships a dead link that every checker passes.** Two
load-bearing SHA pins ride every proposal slice and neither is ever resolution-
checked: the outbox `idea:` blob link (this slice:
`blob/95999b6…/ideas/fleet/schelling-…md`) and the idea file's `> **Grounding:**
<url>@<sha>` line (this slice: `@ab66463`). The LINK pass verifies the PATH
against the working tree and the Grounding regex verifies only the SHAPE —
so a drafter who fat-fingers one hex digit (or pins a branch-local SHA that a
rebase later orphans) publishes a permanently dead URL to sim-lab's pull
surface, and nothing goes red. The fix is hermetic and three lines per pin:
for any `menno420/idea-engine` blob/raw link and any this-repo Grounding pin,
run `git cat-file -e <sha>:<path>` (Grounding: `git cat-file -e <sha>`) in the
local clone at lint time — no network, exit-affecting only for this-repo pins,
foreign-repo pins stay shape-checked as today. Dedup (this slice): distinct
