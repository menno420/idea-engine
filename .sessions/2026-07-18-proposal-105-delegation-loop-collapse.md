## 💡 Session idea
"Random mapping" is a reusable audit lens for any system where each unit points to exactly one other unit chosen without global coordination — one uniform out-edge per node turns the structure into a random functional graph, and its statistics are brutally non-uniform even when every choice was uniform: the cyclic core is O(√N) (a vanishing fraction decides), one giant weakly-connected component swallows ~75.8% of the mass, and the controlling cycle is O(√N). The transferable audit: whenever a design lets each agent delegate/forward/point-to exactly one peer (proxy voting, escalation-to-one, single-upstream trust, one-hop "ask whoever seems best"), do NOT assume uniform choice spreads influence — the emergent concentration is a property of the graph, not of any agent's intent. Candidate future fleet heads: single-upstream escalation chains collapsing onto O(√N) responders, one-retry-target failover concentrating load on a giant component, and single-mentor assignment producing an O(√N) effective-teacher core — each a "one uniform out-edge ⇒ random-mapping concentration" claim with the same √(πN/2) / 0.7582 anchors and a cohort-contrast σ.
