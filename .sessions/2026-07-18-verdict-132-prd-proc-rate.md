# VERDICT 132 mirror — the PRD proc-rate compression: the Warcraft III / Dota 2 Pseudo-Random Distribution (PRD) anti-streak proc system replaces a flat per-attempt proc chance p with an ESCALATING one — the chance starts at a constant increment C and rises by C on every consecutive failure, P(proc on the n-th attempt since the last proc)=min(1,C·n), resetting to C on a proc — which bounds bad-luck streaks (the chance hits 1 by attempt ⌈1/C⌉) and smooths good-luck streaks (why competitive games use it for crit/bash/evasion). The counterintuitive trap: the increment constant C is NOT the proc rate. The inter-proc gap N is a renewal walk with E[N]=Σ n·P(N=n), P(N=n)=[∏_{k<n}(1−min(1,C·k))]·min(1,C·n), and the long-run EFFECTIVE rate is exactly 1/E[N] (renewal-reward) — because the escalating chance is forced toward 1 as the miss streak grows, the walk is SHORT (E[N] sits well below 1/C) so 1/E[N] sits well ABOVE C. For C=0.25 the walk terminates at n=4 (C·4=1.0, a guaranteed proc): P(N=1)=0.25, P(N=2)=0.375, P(N=3)=0.28125, P(N=4)=0.09375, E[N]=2.21875, effective rate=1/2.21875=0.450704 — nearly DOUBLE 0.25. A designer who naively sets C=nominal ("C=0.25 for 25% crit") ships a proc that fires ~80% more often than intended; the fix is to SOLVE C for the target rate (invert 1/E[N](C)=r), which for r=0.25 gives C≈0.0847 (the published Dota 2 PRD constant), a third of the nominal — and what PRD actually buys is a streak bound (⌈1/C⌉−1 misses, at most 3 for C=0.25) not a rate change. ## 💡 Session idea
This slice pins the single-PRD compression on ONE escalating-reset proc in isolation. But competitive games rarely run one PRD proc alone — a hero commonly carries several INDEPENDENT PRD systems at once (crit + bash + evasion + a proc-on-hit item), each with its own increment C_i and its own inter-proc renewal walk. If a designer makes the same naive error on each (C_i=nominal_i), the overshoots do not just persist, they COMPOUND at the "at least one proc this attack" level: the per-attack union proc probability under naive constants is 1−∏_i(1−p_i(streak_i)) evaluated along each system's escalating state, whose long-run realized union rate sits even further above the intended union than any single 1/E[N_i] error would suggest, AND the joint bad-luck-streak structure (the max run with NO proc of ANY kind) tightens in a way no single-system ⌈1/C_i⌉−1 bound captures. A genuinely distinct next GAME slice pins this MULTI-PRD interference as a seed-pinnable head on a hero carrying K=3 independent PRD procs: pre-register (G-a) a union-overshoot gate — the realized union proc rate under naive per-system C_i=nominal_i exceeds the intended union rate 1−∏(1−r_i) by ≥ a floor at ≥3σ (the errors compound, not merely coexist); (G-b) a solve-each control — solving each C_i for its own target r_i collapses the union rate back onto 1−∏(1−r_i) within 3σ, isolating the per-system C=nominal error as the cause, not the multiplicity; and (G-c) a closed-form anchor — the realized union rate matches the product-of-renewal-rates prediction 1−∏(1−1/E[N_i]) within 3σ, so the operator lesson sharpens from "solve C for the target rate" to "solve EACH C for its own target AND verify the joint union/streak behaviour — independent PRD systems interfere at the union level, so a per-system fix is necessary but not sufficient for the felt proc experience." A seed-pinnable multi-system anti-streak head distinct from this slice's single-PRD compression and from the outbox's from-attempt-1 single-escalation frame.
